### Testing
요즘 Test code를 작성하는 데 꽤 많은 시간을 할애하고 있는데,

- Test Context 준비는 어떻게 할 것인가?
- 좋은 Test case는 무엇인가?
- 어떤 상태를 **_정답_**으로 판별할 것인가?

이런 문제들이 참 골치아프다. 문제를 잘 풀어내는 것 만큼이나 어려운 것 같다.

여튼 잘만 해 놓으면 구현을 바꿀 때 마다 수동 테스트 하는 것 보다 훨씬 편한 건 분명한 것 같다.
